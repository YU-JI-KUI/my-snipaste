# my-snipaste

轻量 Windows 截图工具，参考 [Snipaste](https://www.snipaste.com/) 设计。
支持区域截图、多种标注工具、贴图常驻屏幕，专为公司内网设计，无需安装，下载即用。

---

## 快速开始

### 前提：安装 Python 3.8+

1. 下载：https://www.python.org/downloads/
2. 安装时**必须勾选 "Add Python to PATH"**

### 启动

```
git clone https://github.com/YU-JI-KUI/my-snipaste.git
cd my-snipaste
```

根据你的环境选择启动方式：

| 环境 | 启动方式 |
|------|---------|
| 普通 Windows 环境 | 双击 `start.bat` |
| 企业内网（禁止执行 .bat / PowerShell） | 双击 `start.pyw` |

> **为什么 `.pyw` 能绕过限制？**
> `.pyw` 文件由 Windows 自动关联到 `pythonw.exe`，属于普通的文件打开操作，
> 不触发脚本执行策略（BAT/PS1 的限制）。只要 Python 已安装且 `.pyw` 关联正常即可使用。

启动成功后，右下角系统托盘（通知区域）出现相机图标，即可使用。

## 功能说明

### 截图流程

| 操作 | 说明 |
|------|------|
| `Ctrl+Alt+A` | 全局热键，程序在后台时同样生效 |
| 拖拽鼠标 | 框选区域，选区内高亮、选区外变暗 |
| `ESC` / 右键 | 取消截图 |

松开鼠标后自动进入编辑模式，选区边框变蓝，出现 8 个控制柄（Handles）。

### 选区调整（松手后）

| 操作 | 说明 |
|------|------|
| 拖动控制柄 | 调整选区大小（仅 Move 模式下生效） |
| 拖动选区内部 | 移动整个选区（仅 Move 模式下生效） |
| 点击选区外部 | 重新绘制新选区 |

### 工具栏

截图松手后工具栏自动出现在选区正下方，分两行：

**第一行 — 工具 + 操作**

| 按钮 | 说明 |
|------|------|
| ✥ 移动 | 拖动选区或调整控制柄 |
| T 文字 | 点击截图内输入文字，Enter/Tab 确认，文字可拖动位置 |
| → 箭头 | 拖拽绘制带箭头线段 |
| □ 矩形 | 拖拽绘制空心矩形框 |
| ▦ 马赛克 | 拖拽选区打码（可撤销） |
| ↩ 撤销 | `Ctrl+Z`，逐步撤销，马赛克可完全恢复 |
| 📌 贴图 | 将当前截图贴到屏幕常驻显示 |
| 💾 保存 | `Ctrl+S`，弹出文件对话框，默认保存到桌面 |
| ✓ 复制 | `Ctrl+C`，复制到剪贴板后关闭 |
| ✕ 关闭 | 关闭截图 |

**第二行 — 样式（文字/箭头/矩形工具激活时显示）**

- 8 种预设颜色（红/橙/黄/绿/蓝/紫/白/黑）
- 字号调节（8 ~ 72），支持直接输入

### 贴图模式（📌）

点击贴图后，截图以独立浮动窗口常驻屏幕：

| 操作 | 说明 |
|------|------|
| 拖动 | 移动贴图窗口位置 |
| 双击 / 右键 | 关闭贴图窗口 |

可同时贴多张图，互不干扰。

### 系统托盘

右键托盘图标：
- **截图**：等同于按 `Ctrl+Alt+A`
- **退出**：完全退出程序

## 内网环境配置 pip 镜像

公司内网通常无法直接访问 PyPI（pip 默认源），`start.bat` 运行时会卡在依赖安装步骤。解决方法：

**方法一：配置企业内网镜像（推荐，一次配置永久生效）**

询问公司 IT / 运维，获取内网 pip 镜像地址，然后执行：

```bat
pip config set global.index-url http://你的内网镜像地址/simple/
pip config set global.trusted-host 你的内网镜像域名或IP
```

配置完成后正常运行 `start.bat` 即可。

**方法二：临时指定镜像（每次安装时指定）**

打开命令行，手动执行：

```bat
pip install -r requirements.txt -i http://你的内网镜像地址/simple/ --trusted-host 你的内网镜像域名或IP
```

安装成功后再双击 `start.bat`（此时依赖已存在，不会重新下载）。

**方法三：在有外网的机器上提前打包依赖**

```bat
:: 在有外网的机器上执行，把依赖下载到 packages/ 目录
pip download -r requirements.txt -d packages/

:: 把整个项目目录（含 packages/）复制到内网机器，然后执行
pip install -r requirements.txt --no-index --find-links packages/
```

## 常见问题

**Q: 公司不让运行 .bat 文件怎么办？**
双击 `start.pyw` 替代 `start.bat`，效果完全相同，且无控制台黑窗口。
如果双击 `.pyw` 没有反应，右键 → 打开方式 → 选择 `pythonw.exe`（在 Python 安装目录下）。

**Q: 双击 start.bat 提示"依赖安装失败"**
内网环境无法访问外网 pip 源，参考上方「内网环境配置 pip 镜像」章节。`start.pyw` 遇到同样问题时会弹窗提示。

**Q: 双击 start.bat / start.pyw 没有反应或弹出其他错误**
查看同目录下的 `error.log` 获取详细错误信息。

**Q: 快捷键没有反应**
部分安全软件会拦截键盘钩子，将 Python 加入信任列表，或通过右键托盘图标截图。

**Q: 系统托盘没有图标**
说明 pystray 未安装成功，重新运行 start.bat，安装失败时会有错误提示。

**Q: 复制到剪贴板失败**
需要 pywin32，运行 start.bat 会自动安装，失败时请改用保存功能。

**Q: 重复双击 start.bat 启动了多个程序**
程序内置单实例检测，重复启动自动退出，不会重复运行。
